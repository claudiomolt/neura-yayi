# Neura Yayi — Programa 1h: Competencia de monedas

Repositorio privado de investigación y preparación para el programa de 1 hora.

## Tesis (registro analítico)

Tres tendencias en paralelo:

1. **Competencia de monedas estatales** en curso (BRICS, CIPS, petroyuan, monedas locales).
2. **El dólar pierde fuerza relativa** y se defiende con SWIFT, sanciones y poder militar (rutas, energía, chips).
3. **Activos neutrales** (oro físico, Bitcoin digital) capturan demanda de hedge porque son **inconfiscables, libres y con oferta verificable**.

Sin moralizar. Datos duros con fuente.

## Los tres pilares

1. **Dólar / guerra** — Por qué el dólar manda y cómo sostiene la demanda por un token de exportación de déficit fiscal.
2. **BRICS / China** — Competencia de monedas estatales: CIPS, yuan, monedas locales, energía. Todas fiat, emisión infinita.
3. **Oro / Bitcoin / Activo neutralidad soberano** — Activos que no dependen de permiso. Competencia oro vs Bitcoin por la reserva del siglo; oro para Estados, Bitcoin para individuos y red digital.

## Archivos clave

- `index.html` — landing con accesos a ambas presentaciones.
- `presentacion-neura.html` — deck para vivo (**20 slides**), visual, una idea por diapositiva, con navbar e índice expandible.
- `presentacion-extendida.html` — deck de preparación (**35 slides**), con datos, notas para decir en vivo, casos por país/empresa y fuentes.
- `CONTEXT.md` — **estado completo del proyecto, datos verificados, decisiones de estructura, TODOs y guía para continuar desde otra máquina**.
- `MACHETE.md` — versión extendida para estudiar: narrativa, datos, frases y desarrollo oral.
- `PROGRAMA_1H.md` — escaleta de 60 minutos.
- `SLIDE_STORYBOARD.md` — storyboard visual base.
- `SOURCES.md` — fuentes y claims a verificar/actualizar.

## Atajos de teclado (en ambas presentaciones)

- `←` `→` · `Espacio` · `Enter` — navegar
- `1` `2` `3` — saltar al inicio de cada pilar
- `M` — abrir índice · `Esc` — cerrar
- `F` — pantalla completa

## Regla de trabajo

- La versión extendida puede tener texto, datos, narrativa y notas.
- La versión Neura debe quedar entre **10 y 15 slides**, visual, poco texto, una idea por diapositiva.
- Todo claim fuerte debe tener fuente o quedar marcado como `TODO: verificar`.
